ISSUE:
------
When clicking the "View" button in the Dashboard messages table, the modal was 
showing "No message text available" instead of displaying the actual SMS message 
content.

ROOT CAUSE:
-----------
The system was designed for privacy and only stored a hash of the message 
(message_hash) in the database, not the actual message text. The backend API 
was returning message_preview (partial hash) instead of the actual message.

SOLUTION:
---------

1. DATABASE MODEL UPDATE (backend/api/models.py):
   ✅ Added new field: message_text = models.TextField(blank=True, null=True)
   ✅ This field stores the actual SMS message content for viewing later

2. BACKEND API UPDATE (backend/api/views.py):
   
   A. PredictView - Save message text:
      ✅ Updated SMSLog.objects.create() to include:
         message_text=message  # Store the actual message text
   
   B. RecentMessagesView - Return message text:
      ✅ Changed from: 'message_preview': msg.message_hash[:16] + '...'
      ✅ Changed to: 'message_text': msg.message_text if msg.message_text else 'No message text available'

3. DATABASE MIGRATION:
   ✅ Created migration: 0004_smslog_message_text.py
   ✅ Applied migration successfully
   ✅ Database schema updated

4. FRONTEND (frontend/src/components/Dashboard.jsx):
   ✅ Already correctly configured to display message_text
   ✅ Modal shows: {viewingMessage.message_text || 'No message text available'}
