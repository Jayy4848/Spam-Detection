HOW IT WORKS
═══════════════════════════════════════════════════════════════════════════════

FLOW:

1. User opens Home page or About page
2. Component mounts and calls useEffect hook
3. useEffect calls getModelInfo() API function
4. API makes GET request to /api/model-info/
5. Backend reads model_metadata.json file
6. Backend extracts metrics and formats them
7. Backend returns JSON response with model info
8. Frontend receives data and stores in modelInfo state
9. React re-renders with actual model information
10. User sees real accuracy and model name

LOADING STATE:

• While fetching: Shows "..." in hero stats
• After loading: Shows actual accuracy (86.67%)
• If error: Falls back to not showing banner (graceful degradation)

CACHING:

• Data fetched once on component mount
• Stored in component state
• No repeated API calls during session
• Refresh page to reload model info

TECHNICAL NOTES
═══════════════════════════════════════════════════════════════════════════════

• Model metadata is read from: backend/ml_models/trained_models/model_metadata.json
• This file is generated by train_models.py during model training
• Metrics are calculated using scikit-learn's evaluation functions
• Accuracy is test set accuracy (not training accuracy)
• Cross-validation uses 5-fold stratified CV
• All percentages are rounded to 2 decimal places

IMPORTANT:
• If you retrain the model, the displayed metrics will automatically update
• No code changes needed - just retrain and restart backend
• The API reads the latest model_metadata.json file
