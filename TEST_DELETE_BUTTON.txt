🧪 QUICK TEST GUIDE - Delete Button
=====================================

## RESTART BACKEND (IMPORTANT!)
The CORS settings were changed, so you MUST restart the backend:

```bash
# Stop the backend (Ctrl+C if running)
cd backend
python manage.py runserver
```

## START FRONTEND
```bash
cd frontend
npm start
```

## TEST DELETE FUNCTIONALITY

### Step 1: Navigate to Dashboard
- Open http://localhost:3000
- Click "Dashboard" in navigation

### Step 2: Find a Message
- Look at the messages table
- Find any message row

### Step 3: Click Delete Button
- Click the trash icon (🗑️) on the right
- You should see a confirmation dialog

### Step 4: Confirm Deletion
- Click "OK" on the confirmation dialog
- Watch for:
  ✅ Delete button shows loading spinner
  ✅ Message disappears from table
  ✅ Success alert: "✅ Message deleted successfully!"
  ✅ Stats cards update (total count decreases)

### Step 5: Check Console
- Open DevTools (F12)
- Go to Console tab
- Should see NO errors
- Should see successful DELETE request in Network tab

## EXPECTED RESULTS

✅ Confirmation dialog appears
✅ Loading spinner shows during delete
✅ Message is removed from table
✅ Success message appears
✅ Stats are updated
✅ No console errors
✅ No CORS errors

## IF IT DOESN'T WORK

### Check 1: Backend Running?
```bash
curl http://localhost:8000/api/health/
```
Should return: `{"status": "healthy"}`

### Check 2: CORS Headers
- Open DevTools → Network tab
- Try deleting a message
- Find the OPTIONS request (preflight)
- Check Response Headers
- Should see: `Access-Control-Allow-Methods: GET, POST, PUT, PATCH, DELETE, OPTIONS`

### Check 3: Console Errors
Look for:
- ❌ CORS errors → Backend not restarted
- ❌ 404 errors → Wrong URL
- ❌ 405 errors → Method not allowed (CORS issue)
- ❌ Network errors → Backend not running

## QUICK FIX CHECKLIST

- [ ] Backend is running on port 8000
- [ ] Backend was restarted after CORS changes
- [ ] Frontend is running on port 3000
- [ ] No console errors
- [ ] Confirmation dialog appears
- [ ] Delete button shows spinner
- [ ] Message is removed
- [ ] Success message appears

## SUCCESS CRITERIA

When you click delete:
1. ⏱️ Confirmation dialog → Click OK
2. ⏳ Loading spinner appears
3. 🗑️ Message disappears from table
4. ✅ Success alert appears
5. 📊 Stats update automatically
6. 🎉 No errors in console

If all 6 steps work → DELETE IS WORKING! ✅

## TROUBLESHOOTING COMMANDS

### Test Backend Endpoint Directly:
```bash
# Get all messages
curl http://localhost:8000/api/recent-messages/

# Delete a specific message (replace 1 with actual ID)
curl -X DELETE http://localhost:8000/api/messages/1/
```

### Check Backend Logs:
Look at the terminal where backend is running.
Should see:
```
DELETE /api/messages/1/ 200 OK
```

### Check Frontend Network Tab:
1. Open DevTools (F12)
2. Go to Network tab
3. Click delete button
4. Should see:
   - OPTIONS /api/messages/1/ → 200 OK (preflight)
   - DELETE /api/messages/1/ → 200 OK (actual delete)

## COMMON ISSUES

### Issue 1: "CORS policy: Method DELETE is not allowed"
**Solution:** Restart the backend server

### Issue 2: "Failed to delete message"
**Solution:** Check backend is running and message ID exists

### Issue 3: Nothing happens when clicking delete
**Solution:** Check console for JavaScript errors

### Issue 4: Confirmation dialog doesn't appear
**Solution:** Check browser popup blocker settings

After successful delete:
- [ ] Message is gone from table
- [ ] Total messages count decreased by 1
- [ ] Category count updated (if applicable)
- [ ] No errors in console
- [ ] Backend logs show successful DELETE

If all checked → PERFECT! Delete is working! 🎉
