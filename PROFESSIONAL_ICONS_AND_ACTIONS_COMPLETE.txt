4. BACKEND DELETE ENDPOINT
═══════════════════════════════════════════════════════════════════════════════

NEW API ENDPOINT:
─────────────────────────────────────────────────────────────────────────────
DELETE /api/messages/<message_id>/

Request:
• Method: DELETE
• URL: http://localhost:8000/api/messages/123/
• No body required

Response (Success):
{
  "status": "success",
  "message": "Message deleted successfully"
}

Response (Not Found):
{
  "status": "error",
  "message": "Message not found"
}

Response (Error):
{
  "status": "error",
  "message": "Failed to delete message"
}

FEATURES:
• Validates message exists
• Logs deletion in audit log
• Returns appropriate status codes
• Error handling
• Security logging

IMPLEMENTATION:
File: backend/api/views.py
Class: DeleteMessageView(APIView)
Method: delete(request, message_id)

File: backend/api/urls.py
Route: path('messages/<int:message_id>/', DeleteMessageView.as_view())
