═══════════════════════════════════════════════════════════════════════════════
DATA FLOW
═══════════════════════════════════════════════════════════════════════════════

DASHBOARD:
1. Dashboard component mounts
2. useEffect calls getModelInfo() API
3. Backend reads model_metadata.json
4. Backend returns model metrics
5. Frontend stores in modelInfo state
6. React renders Model Performance Card
7. User sees model name and accuracy metrics

RESULT CARD:
1. User analyzes a message
2. Backend runs prediction with multiple models
3. Backend returns result with model_comparison object:
   {
     "model_comparison": {
       "naive_bayes": {
         "category": "spam",
         "confidence": 0.852
       },
       "bert": {
         "category": "spam",
         "confidence": 0.921
       }
     }
   }
4. ResultCard component receives result prop
5. Checks if model_comparison exists
6. Renders Model Predictions Comparison section
7. Shows Naive Bayes prediction
8. Shows BERT prediction (if not "N/A")
9. User sees individual model predictions

TECHNICAL NOTES
═══════════════════════════════════════════════════════════════════════════════

• Model info is fetched once on component mount (not on every render)
• Dashboard card only appears if modelInfo is loaded successfully
• Result card comparison only appears if model_comparison exists in result
• BERT prediction only shows if category is not "N/A"
• All percentages are rounded to 1 decimal place for display
• Confidence values are multiplied by 100 (0.852 → 85.2%)
• Grid layouts use auto-fit for responsive behavior
• Hover effects use transform: translateY() for performance
• Colors use CSS variables for consistency
