🗑️ NEW FEATURE: RESET ALL ANALYTICS DATA

========================================
WHAT WAS ADDED
========================================

1. BACKEND API ENDPOINT
   Route: POST /api/reset/
   Function: ResetDataView
   Action: Deletes all data from database
   
   Deletes:
   - All SMS logs
   - All user feedback
   - All threat patterns
   - All model performance data
   - All sender profiles
   - All analytics snapshots

2. FRONTEND API FUNCTION
   File: frontend/src/services/api.js
   Function: resetData()
   Method: POST request to /api/reset/

3. DASHBOARD BUTTON
   Location: Top-right of Analytics Dashboard
   Style: Red gradient button with trash icon
   Text: "🗑️ Reset Data"
   
4. CONFIRMATION DIALOG
   Shows: "⚠️ Are you sure you want to reset all data?"
   Warning: "This action cannot be undone!"
   
5. SUCCESS/ERROR ALERTS
   Success: "✅ All data has been reset successfully!"
   Error: "❌ Failed to reset data. Please try again."

========================================
HOW IT WORKS
========================================

USER FLOW:
1. User clicks "🗑️ Reset Data" button
2. Confirmation dialog appears
3. User confirms or cancels
4. If confirmed:
   - Button shows "🔄 Resetting..."
   - API call to backend
   - All data deleted from database
   - Success message shown
   - Dashboard reloads with empty stats
5. If cancelled:
   - Nothing happens

BACKEND PROCESS:
1. Receives POST request to /api/reset/
2. Counts records before deletion
3. Deletes all data from 6 tables
4. Logs the reset action in audit log
5. Returns success response with counts
6. If error occurs, returns error response
