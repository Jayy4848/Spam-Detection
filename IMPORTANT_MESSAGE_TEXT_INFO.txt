WHY OLD MESSAGES DON'T SHOW TEXT:
----------------------------------
The system was originally designed for privacy - it only stored a hash of 
messages, not the actual text. When we added the message_text field, existing 
messages in the database don't have this field populated (it's NULL).

This is NOT a bug - it's expected behavior for backward compatibility.

SOLUTION FOR OLD MESSAGES:
--------------------------
If you want to see the text for old messages, you have two options:

Option 1: Analyze them again
   - Copy the message text (if you remember it)
   - Analyze it again on the Home page
   - The new entry will have the text stored

Option 2: Reset all data and start fresh
   - Click "Reset Data" button in Dashboard
   - This will delete all old messages
   - Analyze new messages
   - All new messages will have text stored

TECHNICAL EXPLANATION:
----------------------

Database Schema Change:
```sql
-- Before:
CREATE TABLE sms_log (
    id INTEGER PRIMARY KEY,
    message_hash VARCHAR(64),  -- Only hash stored
    category VARCHAR(20),
    -- ... other fields ...
);

-- After:
CREATE TABLE sms_log (
    id INTEGER PRIMARY KEY,
    message_hash VARCHAR(64),  -- Hash still stored
    message_text TEXT,         -- NEW: Actual text stored
    category VARCHAR(20),
    -- ... other fields ...
);
```

Old messages have message_text = NULL
New messages have message_text = "actual message content"
