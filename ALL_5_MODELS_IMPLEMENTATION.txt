ENSEMBLE VOTING LOGIC
═══════════════════════════════════════════════════════════════════════════════

How the final prediction is determined:

1. Each of the 5 models votes for a category
2. Count votes for each category
3. Category with most votes wins
4. Confidence = average of confidences from models that voted for winner

Example:
• Naive Bayes: spam (85%)
• Logistic Regression: spam (92%)
• Random Forest: spam (88%)
• Gradient Boosting: spam (91%)
• SVM: spam (87%)

Result: spam wins (5 votes)
Final confidence: (85 + 92 + 88 + 91 + 87) / 5 = 88.6%

Example with disagreement:
• Naive Bayes: spam (85%)
• Logistic Regression: spam (92%)
• Random Forest: promotion (78%)
• Gradient Boosting: spam (91%)
• SVM: spam (87%)

Result: spam wins (4 votes vs 1)
Final confidence: (85 + 92 + 91 + 87) / 4 = 88.75%

ACCURACY:
• Ensemble voting typically improves accuracy by 2-5%

TECHNICAL DETAILS
═══════════════════════════════════════════════════════════════════════════════

Model Characteristics:

1. Naive Bayes
   • Fast, simple probabilistic classifier
   • Works well with text data
   • Assumes feature independence
   • Good baseline model

2. Logistic Regression
   • Linear model with sigmoid activation
   • Interpretable coefficients
   • Fast training and prediction
   • Good for linearly separable data

3. Random Forest
   • Ensemble of decision trees
   • Handles non-linear relationships
   • Resistant to overfitting
   • Slower but more accurate

4. Gradient Boosting
   • Sequential ensemble method
   • Builds trees to correct previous errors
   • Often highest accuracy
   • Slower training

5. SVM (Support Vector Machine)
   • Finds optimal decision boundary
   • Works well in high dimensions
   • Effective with TF-IDF features
   • Good generalization
