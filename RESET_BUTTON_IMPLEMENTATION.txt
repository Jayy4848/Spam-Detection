═══════════════════════════════════════════════════════════════════════════════
WHAT DATA GETS DELETED
═══════════════════════════════════════════════════════════════════════════════

The reset operation deletes ALL records from these tables:

1. SMSLog (53 records currently)
   • All analyzed SMS messages
   • Categories, confidence scores, phishing scores
   • Sentiment analysis, urgency scores
   • Behavioral features, timestamps

2. UserFeedback
   • All user corrections to model predictions
   • Original vs corrected categories

3. ThreatPattern
   • All detected threat patterns
   • Pattern types, severity levels, frequencies

4. ModelPerformance
   • All model performance metrics
   • Accuracy, precision, recall scores

5. SenderProfile
   • All sender reputation data
   • Trust scores, message counts

6. AnalyticsSnapshot
   • All historical analytics snapshots
   • Daily/weekly aggregated statistics

═══════════════════════════════════════════════════════════════════════════════
TECHNICAL NOTES
═══════════════════════════════════════════════════════════════════════════════

• The reset operation is ATOMIC (all-or-nothing)
• If any table deletion fails, Django will rollback the entire transaction
• The operation typically completes in < 100ms for small datasets
• No data is backed up before deletion (add backup feature if needed)
• The ML models themselves are NOT deleted (only the analyzed data)
• Trained model files remain intact in backend/ml_models/trained_models/
